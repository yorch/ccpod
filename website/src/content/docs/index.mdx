## What is ccpod?

Claude Code is a powerful agent — but by default it runs on your host with your shell, your network, and your `~/.claude` directory. **ccpod** runs it inside a container instead, with the entire environment described by a *profile*: a versioned YAML file that pins the image, settings, plugins, `CLAUDE.md`, and MCP servers. The agent stays sandboxed, the setup stays reproducible, and the same profile can be shared across machines, teammates, or contributors to an open-source repo.

## 30-second taste

```sh
# Install
curl -fsSL https://ccpod.brnby.com/install.sh | sh

# First-run wizard: detect runtime, set up auth, create default profile
ccpod init

# Drop into Claude in the current directory
ccpod run
```

Want to share an environment with your team? Point a profile at a git repo and everyone gets the same `CLAUDE.md`, settings, and plugin set — synced daily.

```yaml
# ~/.ccpod/profiles/team/profile.yml
config:
  source: git
  repo: https://github.com/your-org/claude-config
  sync: daily
```
